Active Directory — Méthode AGDLP

Objectif : Mettre en place une gestion structurée des droits d'accès aux ressources partagées en entreprise via la méthode AGDLP (Accounts, Global groups, Domain Local groups, Permissions) sous Windows Server.

Réalisations :

  • Création d'une hiérarchie d'Unités d'Organisation (OU) dans Active Directory
  • Mise en place de comptes utilisateurs et de groupes Globaux par service (Comptabilité, RH, Direction, Informatique)
  • Création de groupes Domaine Local avec niveaux d'accès distincts (Lecture, Modification, Contrôle Total)
  • Imbrication des groupes selon la chaîne AGDLP : A → G → DL → P
  • Configuration d'un partage réseau SMB et application des permissions NTFS par groupe Domaine Local
  • Déploiement automatique des lecteurs réseau via stratégie de groupe (GPO) avec ciblage par groupe AD
  • Tests de validation des accès et documentation de la procédure complète

Compétences mobilisées : Administration Active Directory, gestion des droits NTFS/SMB, stratégies de groupe (GPO), structuration des accès réseau, virtualisation Windows Server

Lien référentiel : Bloc 2 — Administrer et sécuriser les ressources d'un domaine Windows

DHCP & DHCP Relay — Linux (Debian)

Objectif : Mettre en place un service DHCP centralisé avec relais DHCP sur des machines virtuelles Debian sous VMware Workstation, permettant la distribution automatique d'adresses IP à travers plusieurs réseaux segmentés.

Réalisations :

  • Déploiement de 3 machines virtuelles Debian 12 sous VMware Workstation (Serveur DHCP, Routeur, Client)
  • Installation et configuration du service isc-dhcp-server avec déclaration des plages d'adresses et options réseau
  • Création et attribution des LAN Segments VMware (SERVEUR, CLIENT) pour isoler les réseaux
  • Configuration du routeur Debian avec 3 interfaces réseau (LAN SERVEUR, LAN CLIENT, NAT)
  • Mise en place du service DHCP Relay (isc-dhcp-relay) pour relayer les requêtes DHCP entre les segments réseau
  • Activation de l'IP Forwarding et configuration du NAT via iptables pour donner accès à Internet aux clients
  • Phase de tests : vérification de l'attribution des adresses IP et de la connectivité Internet depuis le client

Compétences mobilisées : Administration Linux, configuration réseau (interfaces, routage, NAT), service DHCP, DHCP Relay, virtualisation VMware Workstation, segmentation réseau

Attaque MITM SSH & Contre-mesures — Cybersécurité

Objectif : Simuler une attaque de l'homme du milieu (Man-in-the-Middle) sur un service SSH dans un environnement virtualisé sous VMware, puis analyser les vulnérabilités exploitées et mettre en place des contre-mesures adaptées.

Réalisations :

  • Déploiement d'une maquette réseau sous VirtualBox composée de 4 machines virtuelles (serveur SSH Debian, client SSH Debian, attaquant Kali Linux, routeur OpenBSD)
  • Reconnaissance réseau avec nmap pour identifier les hôtes et services exposés
  • Mise en œuvre d'une attaque ARP Spoofing avec ettercap pour intercepter les communications entre le client et le serveur SSH
  • Exploitation de la vulnérabilité via l'outil ssh-mitm et redirection de flux avec iptables pour capturer les identifiants en clair
  • Analyse des trames réseau avec Wireshark pour comprendre le déroulement de l'attaque
  • Mise en place de contre-mesures : authentification par paires de clés ECDSA, désactivation de l'authentification par mot de passe, publication des empreintes via le protocole SSHFP dans la zone DNS
  • Durcissement du service OpenSSH selon les recommandations ANSSI : restriction des algorithmes de chiffrement autorisés, séparation des privilèges, limitation des accès utilisateurs, changement de port

Compétences mobilisées : Cryptographie (symétrique, asymétrique, hachage), exploitation de vulnérabilités ARP, analyse de trames réseau, administration OpenSSH, sécurisation de services réseau, Kali Linux, ethical hacking
